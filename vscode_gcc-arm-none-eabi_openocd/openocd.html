openocd使用笔记

TGL 2021-12-23

  • openocd 安装
  • openocd 使用
    • 命令行基本使用:
    • Telnet 终端骚操作
  • 帮助列表

openocd 是一个开源调试器(Open On-Chip Debugger),官网 www.openocd.org
从openocd 下载页面自行选择所需版本,这儿还提供工具链如 gcc-arm-none-eabi 的下载

openocd 安装

直接解压即可，最好把解压后的目录添加到环境变量中去，这样就可以直接在终端中使用了！
jlink 有可以会出现 openocd 识别不了的情况需要修改 usb 驱动
下图为原来机子上的 jlink 驱图

用 Zadig 修改 jlink 驱动

修改后 openocd 能识别的驱动

注：当驱动切换后 jlink 程序将识别不到 jlink 调试器了，只有将驱重装回 jlink 默认驱动才能识别到。

我从上面的页面下载的 openocd 解压后,drivers/UsbDriverTool.exe”是一个很不错的小工具，它可以切换 jlink 的 usb 驱动.这样当使用 openocd 时选择”BLUCK”驱动，使用 jlink 是选择”jlink”驱动

openocd 使用

命令行基本使用:

  1. 指定 openocd 要驱动的调试器驱动类型
    用openocd -c "adapter driver jlink"指定适配器驱动是 jlink
    可以用openocd -c "adapter list"来查看支持的驱动类型
  2. 指定指定调试器的传输速度
    用openocd -c "adapter driver jlink" -c "adapter speed 4000"指定 jlink,传输速率为 4MHz
  3. 指定调试器跟目标板之间的传输协议
    用openocd -c "adapter driver jlink" -c "adapter speed 4000" -c "transport select swd"指事定 jlin 为 swd 协议

此时虽然可以看到 openocd 已跟 jlink 连上了,但有 1 个警告说 gdb 服务需要指定目标芯片才行。
openocd -c "adapter driver jlink" -c "transport select swd" -f "target/stm32f1x.cfg"
可以完成目标板指定。
“target/stm32f1x.cfg”是 openocd 安装目录 share/openocd/scripts/的文件，该文件指定了 stm32f103c8 的内存分布以及调试器的速度为 1MHz 所的命令行就可以不用指定传输速率了该命令指行结果如下
